# Smart-Farmland-Management-System-using-Spring-Cloud-and-Vue
智慧农田管理平台主要分为农场主页系统公告通知、农场统计分析、农田管理、农作物资管理、系统管理五大业务模块，全平台嵌入通用智能问答功能，为所有操作人员提供农事、业务咨询支撑。平台使用者分为管理员和农场员工两类，系统依据业务范围与操作权限完成角色划分： 管理员：拥有系统全部功能操作权限，可完成系统自定义配置、全模块数据管理，同时可使用智能问答查询系统运维、权限配置相关操作指引； 农场员工：按岗位职责细分采购人员、销售人员、农田管理人员，仅开放对应岗位业务模块，可通过智能问答咨询作物种植、农资调度、产销规划等各类农事业务问题。

## ✨ 功能特性

### 核心业务模块
- **农田环境管理**：支持农田信息增删改查，记录空气温湿度、土壤pH值、光照强度等环境监测数据，支持 Excel 批量导入导出
- **农作物资管理**：完整覆盖农资采购入库、库存台账、农作物销售出库全流程，实现物资全生命周期可追溯管控
- **系统权限管理**：基于 RBAC 权限模型，划分管理员、农场员工两类账号，细分采购人员、销售人员、农田管理员等岗位权限，实现动态菜单加载
- **公告与统计分析**：系统公告发布与管理、农场经营数据与环境监测数据可视化统计，辅助生产经营决策
- **文件管理**：支持图片、Excel 等文件上传与下载，统一返回文件访问地址

### AI 智能能力
- **农事 RAG 知识库问答**：内置农业专业知识库向量检索，精准解答种植技术、病虫害防治、水肥管理等专业问题
- **多轮对话记忆**：基于会话 ID 维护对话上下文，支持连续追问，交互体验更连贯自然
- **业务智能分析**：自动读取农田、采购、销售全量业务数据，生成种植经营分析报告与优化建议
- **农资营销文案生成**：根据农资商品信息自动生成宣传文案
- **熔断降级机制**：大模型服务异常、知识库加载失败时自动降级，保障平台基础功能稳定可用

## 🛠️ 技术栈

### 后端微服务栈
- **核心框架**：Spring Boot 2.7.18（业务模块）、Spring Boot 3.2.0（AI 模块）、Spring Cloud 2021.0.9

## 🏗️ 系统架构

系统采用微服务分层设计，共包含 5 个核心服务模块，各服务职责独立、可单独部署与扩展：

| 服务名称 | 端口 | 核心职责 |
| :--- | :--- | :--- |
| eureka-server | 8761 | 服务注册与发现中心，统一管理所有服务实例 |
| gateway | 8080 | API 统一网关，负责请求路由转发、负载均衡、全局跨域处理 |
| smart-auth | 8081 | 用户认证服务，处理登录注册、JWT 令牌签发校验、权限管理 |
| smart-serve | 8082 | 核心业务服务，承载农田、物资、公告、统计、文件等业务功能 |
| ai-service | 8087 | AI 智能服务，提供多轮对话、RAG 知识库问答、业务数据分析能力 |

整体架构遵循高内聚、低耦合原则，前后端通过 RESTful API 交互，服务间通过 Feign 实现远程调用，支持熔断降级保障系统稳定性。
